# Merge


## *3-way-merge* : 
  - 새로운 브랜치가 생김
  - 새로운 커밋이 생김
  - 브랜치의 범위가 확장
  - 충돌이 발생한 경우
    - 충돌이 발생한 파일을 확인하고
    - 충돌을 수동으로 해결하고
    - 스테이징 에어리어에 수동으로 추가한 후
    - 새로운 커밋을 수동으로 생성 

## *fast-forward-merge* : 
  - 새로운 브랜치가 생기지 않고, 브랜치 범위만 변경
  - 새로운 커밋이 생성되지 않는다.
  - main 브랜치의 범위가 sub 브랜치의 범위와 같아진다.

- git merge sub : master 브랜치에서, 베이스와 sub 브렌치의 차이를 워킹디렉토리와 SA에 복사하고, 커밋.
- 충돌이 발생할경우, 충돌이 발생한 파일을 수동으로 수정하고 add 후 commit
